# Week 02 – Requirement Analysis

## Project Title

**CyberGuard – Password Security Toolkit**

---

## Functional Requirements

The system should allow users to:

* Register a new account.
* Login to the system.
* Check password strength.
* Generate a strong password.
* Detect common weak passwords.
* View password history.
* Read password security tips.
* Manage user profile.
* View activity logs.
* Export scan reports as PDF.

---

## Non-Functional Requirements

The system should:

* Be easy to use.
* Have a simple and responsive user interface.
* Store user data securely.
* Provide fast response time.
* Run on modern web browsers.

---

## User Requirements

Users should be able to:

* Create an account.
* Login securely.
* Analyze password strength.
* Generate secure passwords.
* View previous scan history.
* Export password reports.

---

## System Requirements

### Software

* Ubuntu Linux
* Python
* Flask
* SQLite
* Visual Studio Code
* Google Chrome

### Hardware

* Computer or Laptop
* Minimum 4 GB RAM
* Internet connection (for development)

---

## Project Scope

This project focuses on password security. It provides tools to help users create stronger passwords, detect weak passwords, and improve password security awareness. The system is developed for educational purposes and basic cybersecurity learning.

---

## Week 02 Progress

During Week 02, the project requirements were analyzed. Functional requirements, non-functional requirements, user requirements, system requirements, and project scope were prepared. These requirements will be used during the system design and development stages.
